Buttons

The button component supports three different button variations, Primary button, secondary button and link button. Each variation can be used in two different sizes and together with icons.

Note, if using an input add the HTML attribute type=button or type=submit to the button element. If using a <span> element, remember to add tabindex to the HTML element so that users can navigate to the button with the keyboard.

Primary Button

The Primary button should be the focus of the user’s attention, and work as a call to action. It can for example be places at the end of a form as a submit button.

Note, It’s not supposed to be used next to any other primary button.

Example

Link tag Span tag
  • code
<input type="submit" class="btn btn--primary"/>

<button class="btn btn--primary">Button</button>

<a href="#" class="btn btn--primary">Link tag</a>

<span class="btn btn--primary" tabindex="0">Span tag</span>

Secondary button

The Secondary Button should not be to focus of the users attention and can therefore be used next to other secondary buttons and next to a primary button

Example

Link tag Span tag
  • code
<input type="submit" class="btn btn--secondary"/>

<button class="btn btn--secondary">Button</button>

<a href="#" class="btn btn--secondary">Link tag</a>

<span class="btn btn--secondary" tabindex="0">Span tag</span>

Large button

Larger button

Example

  • code
<button class="btn btn--primary btn--large">Button Large</button>

Button link

Button link can be used as a link or an action, it can for example be used on a card to communicate that there is more information to show.

Example

  • code
<button class="btn btn--link">Button link</button>

Small button

Example

  • code
<button class="btn btn--primary btn--small">Button Small</button>

Disabled button

When a button is disabled it cannot not be clicked

Example

  • code
<button class="btn btn--primary" disabled>Disabled</button>

Button with Icon

Buttons with icon and text support.

Note that you must include classes for the icon element for proper alignment and spacing. Source order matters!

Example

  • code
<!-- Icon only -->
<button class="btn btn--primary"><i class="icon icon--account btn__icon"></i></button>

<!-- Icon on the right -->
<button class="btn btn--primary">
    Primary
    <i class="icon icon--account btn__icon btn__icon--right"></i>
</button>

<!-- Icon on the left -->
<button class="btn btn--secondary">
    <i class="icon icon--menu btn__icon btn__icon--left"></i>
    Secondary
</button>

<!-- Small button with icon -->
<button class="btn btn--secondary btn--small">
    <i class="icon icon--menu btn__icon btn__icon--left"></i>
    Secondary
</button>

Show more button

Show more button for content expanding and collapsing. Toggle between the classes .icon--angle-up and .icon--angle-down depending on the state. Also make sure to change the text.

Example

  • code
<button class="btn btn--show-more">
    <span>Show more</span>
    <i class="icon icon--angle-down btn__icon btn__icon--right"></i>
</button>

<button class="btn btn--show-more">
    <span>Show less</span>
    <i class="icon icon--angle-up btn__icon btn__icon--right"></i>
</button>
